# Sprint 3: 자료 해설형 아카이브 1차 구현

## 스프린트 목표

홍보자료 목록과 상세 페이지를 단순 게시글 목록에서 “읽는 순서가 있는 자료 해설 아카이브”로 발전시킨다. 초기에는 수동 추천과 mock 해설 데이터를 활용해 운영 부담을 낮춘다.

## 기간

- 1주
- Codex 기준 작업 속도: 자료 목록 UI 개선, 상세 페이지 해설 영역, 추천 자료 mock 연결, 검증까지 포함한다.

## 작업 범위

- 제안 2: 자료 해설형 아카이브
- 제안 1의 주제 허브와 연결되는 탐색 링크를 고려한다.
- 제안 3 FAQ/공개 질의 기반 안내 시스템은 제외한다.

## 주요 산출물

- `/materials` 목록의 아카이브형 UI 개선
- 자료 난이도 또는 읽기 단계 mock 표시
- 자료 상세 페이지에 해설 영역 추가
- “이 자료와 함께 볼 자료” 수동 추천 영역 추가
- 영상 콘텐츠 안내와 자료 상세 간 연결 UI

## Codex 작업 단위

1. 현재 `/materials`와 `/materials/[slug]` 구조 확인
2. `site-content.ts` 또는 새 helper에 자료 해설 mock 데이터 추가
3. 자료 목록에 단계/난이도/추천 순서 표시
4. 자료 상세에 해설 요약, 읽기 포인트, 함께 볼 자료 영역 추가
5. 관련 자료가 없는 경우 빈 상태 문구 추가
6. 주제 상세 페이지에서 자료 아카이브로 이동하는 링크 점검
7. `npm run lint`, `npm run build`, Playwright 검증

## 완료 기준

- 자료 목록이 단순 최신순 게시판보다 아카이브 느낌을 준다.
- 자료 상세 페이지에 해설 요약과 추천 자료 영역이 표시된다.
- 추천 자료 데이터가 없어도 화면이 깨지지 않는다.
- 모바일에서 제목, 요약, 추천 카드 텍스트가 잘리지 않는다.
- `npm run lint`와 `npm run build`가 통과한다.

## 제외 범위

- 파일 업로드
- 관계형 추천 테이블
- 관리자 추천 자료 편집 UI
- 영상 실제 업로드/재생 기능
- FAQ 또는 공개 문의 콘텐츠 기능

## 리스크와 대응

- 리스크: 추천 자료를 수동 mock으로 관리하면 실제 게시글과 어긋날 수 있다.
- 대응: slug 기반으로 연결하고, 없는 slug는 표시하지 않도록 방어 로직을 둔다.

- 리스크: 자료 해설 문구가 사실 단정처럼 보일 수 있다.
- 대응: 확정되지 않은 설명은 “읽기 안내”, “자료 이용 순서” 중심으로 표현한다.
