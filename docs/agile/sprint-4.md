# Sprint 4: 주제 허브와 자료 아카이브 통합 안정화

## 스프린트 목표

제안 1과 제안 2의 구현 결과를 하나의 탐색 경험으로 통합한다. 홈, 주제 상세, 자료 목록, 자료 상세 사이의 이동 흐름을 정리하고, 운영자가 혼자 관리하기 쉬운 최소 구조로 안정화한다.

## 기간

- 1주
- Codex 기준 작업 속도: 통합 수정, 회귀 검증, 문서 정리, UI QA까지 포함한다.

## 작업 범위

- 제안 1: 연구 주제 허브 안정화
- 제안 2: 자료 해설형 아카이브 안정화
- 제안 3 FAQ/공개 질의 기반 안내 시스템은 제외한다.

## 주요 산출물

- 홈, 주제 허브, 자료 아카이브 간 CTA 정리
- 중복 문구와 중복 카드 정리
- 빈 상태, 로딩 실패, 데이터 부족 상황 점검
- 관리자 운영 관점의 간단한 문서 메모
- 주요 페이지 Playwright 검증 결과 정리

## Codex 작업 단위

1. 홈, `/topics`, `/topics/[slug]`, `/materials`, `/materials/[slug]` 흐름 점검
2. 중복되거나 혼란스러운 CTA 제거
3. 주제와 자료 간 상호 링크 추가 또는 정리
4. 빈 상태 UI와 오류 가능성 점검
5. `docs/sitemap.md`가 비어 있다면 최소 사이트맵 초안 작성 여부 검토
6. `npm run lint`, `npm run build`
7. Playwright로 홈, 주제, 자료 목록, 자료 상세의 데스크톱/모바일 확인

## 완료 기준

- 방문자가 홈에서 주제 또는 자료 탐색으로 자연스럽게 이동할 수 있다.
- 주제 상세에서 관련 자료를 찾을 수 있다.
- 자료 상세에서 주제 또는 추천 자료로 이동할 수 있다.
- 게시글 데이터가 부족해도 빈 상태 UI가 안정적이다.
- 공개 페이지 주요 경로가 모바일과 데스크톱에서 깨지지 않는다.

## 제외 범위

- FAQ 또는 공개 문의 콘텐츠 기능
- 신규 회원 기능
- 댓글
- 후원 결제
- 파일 업로드
- 관리자 권한 분리

## 리스크와 대응

- 리스크: 탐색 경로가 많아져 사용자가 혼란스러울 수 있다.
- 대응: 홈 CTA는 주제 허브와 자료 아카이브 중심으로 제한한다.

- 리스크: 기능은 늘었지만 운영 문서가 부족할 수 있다.
- 대응: 최종 스프린트에서 운영자가 어떤 데이터를 어디서 관리하는지 짧게 문서화한다.
